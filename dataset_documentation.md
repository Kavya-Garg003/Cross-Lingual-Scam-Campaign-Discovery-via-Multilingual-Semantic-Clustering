# Spam Dataset Processing & Generation Log

This document serves as the official tracking log for the transformation of our initial SMS dataset into the final, multilingual scam corpus. It outlines every processing step, the rationale behind it, and the data statistics at each phase.

---

## 1. Initial State & Analysis
*   **Original File:** `final_scam_dataset.csv`
*   **Rows:** 84,087 total entries.
*   **Initial Balance:** 43,506 Ham (Label 0) vs. 40,581 Spam/Scam (Label 1).
*   **Observations:** The dataset was English-heavy, but standard language detectors (like `langdetect` or `langid`) struggled with Romanized Indian text, severely misclassifying **Hinglish** SMS templates as Swahili (`sw`), Indonesian (`id`), and Dutch (`nl`).

---

## 2. Step-by-Step Processing Pipeline

### Step 1: Ham Deletion & Label Stripping
**Action:** Removed all messages with Label 0 (Ham) and subsequently dropped the `label` column entirely.
**Why:** The training objective shifted to analyzing purely malicious/spam messages. Since 100% of the dataset was now spam, a label column became redundant.
**Resulting Stats:** 40,581 rows remaining.

### Step 2: Custom Indian/Hinglish Language Detection
**Action:** Wrote and executed `detect_indian.py`. This script utilized regex matching against Indian Unicode blocks (Devanagari, Tamil, etc.) and a custom vocabulary set (e.g., *hai, ki, mein, karo*) to force-identify romanized transliterations.
**Why:** To fix the extreme misclassifications caused by standard libraries. 
**Resulting Stats:** Correctly reclassified 267 mislabeled African/European strings into `hinglish`. (Noted 0 usage of actual native Unicode scripts by the scammers in this dataset).

### Step 3: Extreme Deduplication
**Action:** Dropped all rows with exact matching text and aggressively randomized (shuffled 4 times) the remaining rows.
**Why:** Spam datasets are heavily templated, leading to huge clusters of identical texts. Exact duplicates cause machine learning models to overfit rapidly. Shuffling was done to destroy homogenous "clumps" of similar templates.
**Resulting Stats:** 
*   **15,170** exact duplicates identified and utterly removed.
*   **25,411** unique, highly-heterogenous rows remained.

### Step 4: Text Cleaning Regex
**Action:** Applied a strict `clean_text()` python function across the entire corpus.
**Why:** To normalize the data for NLP tokenization. 
*   *(What we stripped):* Multiple whitespaces, weird invisible unicode tokens.
*   *(What we kept):* Standard text, numbers, URLs, and file paths (colons, slashes). Maintaining URLs and numbers was strictly vital, as they serve as the core feature engineering targets for detecting "scam" behavior.

### Step 5: Dataset Segmentation for Translation
**Action:** Splitting the 25,411 clean rows into 5 perfectly even chunks (~5,082 rows per chunk). Outputted as 5 temporary `.csv` files.
**Why:** Python-based free translation APIs trigger IP suspensions when bulk translating massive volumes. By chunking them, they were prepped for a zero-code Google Sheets workaround.

### Step 6: Multilingual Translation (Google Sheets)
**Action:** Uploaded the chunked CSVs into Google Sheets and utilized `=GOOGLETRANSLATE()` to securely format the datasets into 5 language buckets: English, Hindi, Kannada, Tamil, and French.
**Why:** Google’s internal sheet engine processes massive translation volumes entirely for free without tripping API blocks.

### Step 7: Final Post-Processing & Merging
**Action:** Re-ingested the 5 translated Google Sheets exports. Cleaned out artifact columns generated by Sheets (like `Unnamed 0`), hardcoded the respective language target code, slammed them back into a single dataframe, strictly randomized the merge, and saved the result.
**Why:** To finalize the corpus into an ML-ready structure.

---

## 3. Final Dataset Delivery
*   **Final File:** `final_scam_dataset.csv`
*   **Total Rows:** 25,411 uniquely sourced entries.
*   **Format:** 2 Columns (`text`, `language`).
*   **Distribution:** Equitably divided into 5 target languages (`en`, `hi`, `kn`, `ta`, `fr`).

*This curated dataset is now primed for multilingual NLP model fine-tuning and spam-classification analysis!*
